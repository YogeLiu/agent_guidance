# Week 01 — Agent Loop 从零实现

## 本周目标

不用任何 Agent 框架,亲手实现一个可运行的 Agent Loop,搭建工具系统的核心骨架(统一接口、注册表、Feature Flag、HITL)。

> [!tip] 核心原则
> 这周禁用 LangChain / AutoGen / CrewAI。所有代码基于 LLM provider SDK 直接调用(Anthropic 或 OpenAI 协议任选,W2 会抽 `LLMClient` 统一接口)。

> [!info] 本周不做
> `run_python` 工具(放到 W2 周一,与其他高风险工具一并处理)、工具沙箱(放到 W6)、Reflection / 多 Agent / Memory(后续周)。

---

## 阶段0:Agent Loop 跑起来(周一~周三)

**核心能力**:Tool Calling · 流式输出 · 错误恢复

### 周一 04-20

- [ ] 注册 LLM API Key(SiliconFlow / Anthropic / OpenAI 任选),跑通第一个 `chat.completions.create` 调用
- [ ] 实现最小 Agent Loop:用户输入 → LLM 调用 → 打印输出
- [ ] 加流式输出(`stream=True`),用户能看到逐字回复

### 周二 04-21

- [ ] 实现 `web_search` 工具(Tavily API)
- [ ] 实现 `read_file` 工具
- [ ] Loop 检测 `tool_calls` → 执行工具 → 结果作为 `role=tool` 注入 → 再次调用 LLM,直到无 tool_call

### 周三 04-22

- [ ] 重试逻辑:RateLimitError / TimeoutError 指数退避(2^n + jitter, 上限 30s, 最多 3 次)
- [ ] 工具报错结构化注入:统一 `make_tool_error(tool, error_type, message)` 返回 JSON,LLM 据此自我纠正
- [ ] 端到端测试:"帮我搜索 LangGraph 并总结" 全程自动完成

### 阶段0 验收

> [!success] 通过标准
> - web_search 触发 → 结果注入 → LLM 基于结果回答 ✓
> - 工具故意报错 → Agent 在下一轮自我纠正 ✓
> - 全程流式,无黑盒等待 ✓
> - 代码里 **零** Agent 框架依赖 ✓

## 阶段1 启动:Tool System(周四~周日)

**核心能力**:Tool Permission Model · Pydantic Schema · Feature Flag · HITL

### 周四 04-23

- [ ] 定义统一工具接口 `BaseTool`
  - `name` / `description` / `InputSchema(Pydantic)`
  - `execute(args: dict) -> str`
  - `needs_permission() -> "allow" | "deny" | "ask_user"`
  - `to_llm_schema()`:转 OpenAI function-calling 格式
- [ ] 把 `web_search` 和 `read_file` 重构到新接口

### 周五 04-24

- [ ] 新增 `write_file` 工具(`needs_permission = "ask_user"`)
- [ ] HITL:执行前在终端弹确认(展示完整路径与内容长度),拒绝后通过 `make_tool_error` 优雅返回,LLM 能改用其他方式

### 周六 04-25

- [ ] 实现 `ToolRegistry`
  - `register(tool)`:同名重复注册抛 `ValueError`
  - `get(name)`:取工具,未注册抛 `KeyError`
  - `get_tools_for_llm(disabled: set[str] | None) -> list[dict]`:返回当前可见工具的 schema
  - `dispatch(tool_call) -> str`:解析 args、查权限、执行、捕异常,全部走 `make_tool_error` 包装

### 周日 04-26

- [ ] 实现 `FeatureFlags`(**黑名单语义** + 默认开启)
  - `from_env()`:读 `DISABLED_TOOLS=write_file,run_python` 列出要关闭的工具
  - `disabled_set() -> set[str]`:传给 `ToolRegistry.get_tools_for_llm`
  - 设计原则:**新增工具默认对 LLM 可见,危险工具显式关闭**
- [ ] 整体联调:跑 5 条手测 query 覆盖 web_search / read_file / write_file 拒绝场景
- [ ] 写 README:工具接口规范 / permission 三态 / 新增工具的 5 步操作清单
- [ ] **预留 W2 接口**:在仓库根新建空目录 `evals/`,W2 周六会在此建立 Eval Harness 与 baseline
