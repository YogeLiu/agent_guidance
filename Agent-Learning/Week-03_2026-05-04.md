# Week 03 — Memory System

← [[Agent-Learning/Week-02_2026-04-27|Week 02]] · [[Agent-Learning/INDEX|总览]] · [[Agent-Learning/Week-04_2026-05-11|Week 04]] →

## 本周目标

实现工业级记忆系统。不是"把对话历史全量塞入上下文"，而是分类存储、按需检索、自动老化。

> [!info] 设计参考
> 参照 Claude Code 泄露源码中 `memoryTypes.ts` 的四类记忆架构。记忆的核心原则：**只存不可从代码/项目状态推导的信息**。

---

## 四类记忆设计（周一~周二）

### 周一 05-04

- [ ] 定义四类记忆结构

| 类型 | 作用 | 保存时机 |
|---|---|---|
| `user` | 用户角色、偏好、知识 | 了解用户身份时 |
| `feedback` | 行为指导（纠正+确认） | 用户纠正或确认非显而易见方法时 |
| `project` | 项目上下文、目标、约束 | 了解谁在做什么、为什么时 |
| `reference` | 外部系统指针 | 得知外部资源位置时 |

- [ ] 实现记忆文件结构：frontmatter + body（rule → **Why:** → **How to apply:**）
- [ ] 实现 `MEMORY.md` 索引：每条 < 150 字，始终加载到上下文

### 周二 05-05

- [ ] 实现记忆两层分离
  - Index（`MEMORY.md`）：始终注入上下文，轻量
  - Payload（独立 `.md` 文件）：按需读取，避免上下文膨胀
- [ ] 实现记忆写入决策逻辑：对话结束时 Agent 判断哪些信息值得记忆

---

## 记忆生命周期（周三~周四）

### 周三 05-06

- [ ] 实现记忆老化：每条记忆记录 `last_accessed`，超过 30 天未访问标记为低优先级
- [ ] 实现记忆验证：Agent 引用记忆前，先验证对应文件/资源是否仍然存在
- [ ] 实现记忆去重：写入前检查是否存在相似记忆，避免重复

### 周四 05-07

- [ ] 实现记忆合并：Index 超过 50 条时，Agent 自动合并重复/相关条目
- [ ] 实现 AutoDream 雏形：对话结束后扫描本次对话，提取值得记忆的内容
- [ ] 实现三层作用域：`user`（全局 `~/.researchagent/memory/`）· `project`（项目级）· `local`（本地，不共享）
