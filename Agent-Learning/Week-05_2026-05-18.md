# Week 05 — Multi-Agent 基础

← [[Agent-Learning/Week-04_2026-05-11|Week 04]] · [[Agent-Learning/INDEX|总览]] · [[Agent-Learning/Week-06_2026-05-25|Week 06]] →

## 本周目标

实现 Coordinator/Worker 编排模式。重点是**角色分工**和**Worker Prompt 工程**，而不是让多个 Agent 平等对话。

> [!info] 核心设计原则（来自 Claude Code 源码）
> 1. **任务分阶段**：Research → Synthesis → Implementation → Verification
> 2. **并行是超能力**：只读任务自由并行，写任务同文件集串行
> 3. **Worker Prompt 必须自包含**：Worker 看不到 Coordinator 的对话历史
> 4. **合成理解不可委托**：Coordinator 必须自己理解研究结果

## Coordinator 实现（周四）

### 周四 05-21

- [ ] 实现 Coordinator Agent：
  - Research Phase：派发多个 SearcherWorker **并行**执行
  - Synthesis Phase：Coordinator **自己**阅读结果并生成写作提纲（不可委托）
  - Implementation Phase：派发 WriterWorker 写各章节（**串行**，避免风格不一）
  - Verification Phase：Spawn 新 Worker 验证内容准确性

- [ ] 实现 Continue vs Spawn 决策

```python
def should_continue_or_spawn(task, context_overlap):
    if context_overlap > 0.7:
        return "continue"   # 高上下文重叠，继续同一 Worker
    else:
        return "spawn"      # 低重叠/验证任务，新开 Worker
```
