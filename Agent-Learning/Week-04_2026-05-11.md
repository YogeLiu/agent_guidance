# Week 04 — Planning System

← [[Agent-Learning/Week-03_2026-05-04|Week 03]] · [[Agent-Learning/INDEX|总览]] · [[Agent-Learning/Week-05_2026-05-18|Week 05]] →

## 本周目标

Agent 能对复杂任务生成结构化计划，并在执行中动态修正。引入 LangGraph 实现状态机。

> [!warning] 本周引入 LangGraph
> 这是整个学习路径中第一次使用框架。引入原因：状态机是 Planning 的自然实现载体，LangGraph 的 StateGraph 设计和工业级 Agent 几乎同构。

---

## Planner-Executor 分离（周一~周二）

### 周一 05-11

- [ ] 安装 LangGraph，理解 `StateGraph` 核心概念
- [ ] 定义 Plan 数据结构（JSON Schema）

```json
{
  "plan_id": "uuid",
  "goal": "...",
  "tasks": [
    {
      "id": "t1",
      "description": "...",
      "depends_on": [],
      "status": "pending",
      "token_budget": 1000,
      "result": null
    }
  ],
  "total_token_budget": 8000,
  "tokens_used": 0
}
```

- [ ] 实现 Planner：接收用户目标 → 输出结构化 JSON 计划

### 周二 05-12

- [ ] 实现 Executor：按序执行 Plan 中的任务，更新状态
- [ ] 用 LangGraph `StateGraph` 建模：`Plan` 作为 State，节点为执行步骤
- [ ] 实现任务依赖：`depends_on` 字段控制执行顺序

## 动态 Replanning（周三）

### 周三 05-13

- [ ] 实现失败检测：任务失败时不直接报错，而是把失败上下文反馈给 Planner
- [ ] 实现 Replanning：Planner 基于失败原因生成修订版计划（换策略 or 跳过步骤）
- [ ] 测试场景：`web_search` 超时 → Replanner 换关键词重试 or 标记为"数据不足"继续

## Token 预算系统（周四）

### 周四 05-14

- [ ] 实现 Task 级 Token 预算：每个任务分配 `token_budget`
- [ ] 实现预算监控：任务执行中追踪 Token 消耗，超 80% 时触发警告
- [ ] 实现优雅降级：预算用尽时 Agent 主动切换到摘要模式（不再展开详细分析）
- [ ] 整体 Plan 预算：总 Token 上限，临近时强制结束并输出当前进度

## 阶段3 验收

> [!success] 通过标准
> - "搜索 5 个竞品并生成报告" → Agent 先出计划，执行过程可见，报告可读 ✓
> - 某步超时 → 自动 Replan（换策略 or 跳过），不崩溃 ✓
> - Token 用到 80% → 弹出警告并切换摘要模式 ✓
> - 任务中断重启 → 从中断处继续，不重头来 ✓
