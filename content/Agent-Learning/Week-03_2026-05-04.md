# Week 03 — Memory System(数据驱动必要性验证)

## 本周目标

实现工业级记忆系统的**核心闭环**:四类记忆 + Index/Payload 两层 + 跨会话持久化 + 写入决策。**用 10 次真实研究运行验证 4 类记忆是否都真的需要**(任一类 0 次写入 → 砍或重设计)。

> [!info] 设计参考
> 参照 Claude Code `memoryTypes.ts`。**但我们不盲信抽象** —— 周日必须用数据证明 4 类都被用上,否则砍。

> [!warning] 本周聚焦核心
> 老化、合并、AutoDream、三层作用域降级为延伸目标(选做)。先把核心闭环跑通 + 用数据证明设计合理。

## 四类记忆数据结构 + 结构化日志(周一~周二)

### 周一 05-04 — 类型定义 + structlog 接入

- [ ] 定义四类记忆

| 类型 | 作用 | 保存时机 |
|---|---|---|
| `user` | 用户角色、偏好、知识 | 了解用户身份 / 习惯 |
| `feedback` | 行为指导(纠正 + 确认) | 用户纠正或确认非显然方法 |
| `project` | 项目上下文、目标、约束 | 了解谁在做什么、为什么 |
| `reference` | 外部系统指针 | 得知外部资源位置 |

- [ ] 单条记忆文件结构(frontmatter + body)
  ```markdown
  ---
  name: ...
  description: ...   # 一行,用于 Index 检索
  type: user | feedback | project | reference
  created: 2026-05-04
  ---

  正文(feedback / project 必须包含 **Why:** 与 **How to apply:**)
  ```
- [ ] `MEMORY.md` 索引规范:每行 `- [Title](file.md) — 一行 hook`(< 150 字)
- [ ] **接入 `structlog`(系统能力)** —— LLM / Tool / Memory 三个模块全部走统一 logger,带 `session_id` / `step_id`,**一个 session 的所有事件一条 grep 命令拉出**

### 周二 05-05 — Index/Payload + 写入决策

- [ ] 两层分离
  - **Index**:`MEMORY.md`,始终注入到对话 system prompt 最前面
  - **Payload**:独立 `.md`,Agent 显式 `read_memory(name)` 才加载
- [ ] 写入决策:对话结束时 Agent 自评"本次对话是否产生了值得保存的 user/feedback/project/reference",4 类各自有触发条件
- [ ] **关键**:每次写入打 structlog 日志 `event=memory_write type=<user|feedback|project|reference> name=<...>` —— 周日数据分析的依据

## 必要性验证 + Eval 扩展 + W3 baseline(周日)

### 周日 05-10 — 数据驱动决策

- [ ] **跑 10 次真实研究任务**(自选 10 个真实想知道的研究主题),全程 structlog 落盘
- [ ] **统计 4 类记忆写入次数**(grep memory_write 事件分组计数),写到 `docs/w3_memory_usage_report.md`
  ```
  user      : XX 次
  feedback  : XX 次
  project   : XX 次
  reference : XX 次
  ```
- [ ] **决策**(写到本周回顾)
  - 任一类 0 次写入 → **砍掉这一类**,或重新设计触发条件后下周补
  - 4 类都 > 0 次 → 保留,继续 W4
- [ ] `evals/cases.jsonl` 加 3 条记忆用例(跨会话偏好、失效引用、feedback 应用)
- [ ] **跑 W3 baseline**:`python evals/baseline.py --week w3` → `evals/baseline/w3.json`
- [ ] **对比验证**:`python evals/baseline.py --compare w2` —— 必须满足:
  - 对相同主题第二次研究,**token 较 W2 减少 ≥ 30%**(证明 memory 真的减少了重复搜索)
  - LLM-as-Judge 分数不低于 W2 baseline
  - 否则:周日复盘必须写"memory 没起作用,原因是什么,W4 怎么改"
