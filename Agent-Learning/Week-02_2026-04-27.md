# Week 02 — Tool System 完整落地

← [[Agent-Learning/Week-01_2026-04-20|Week 01]] · [[Agent-Learning/INDEX|总览]] · [[Agent-Learning/Week-03_2026-05-04|Week 03]] →

## 本周目标

完成工具系统的全部能力：权限模型、Schema 验证、Feature Flag、错误恢复、Human-in-the-Loop 闭环测试。

> [!warning] 本周不新增工具
> 重点是把现有工具系统打磨到生产可用，而不是堆砌更多工具。

---

## 阶段1 收尾：Tool System 完整（周一~周三）

**核心能力**：工具权限 · 错误处理规范 · 工具并行调用

### 周一 04-27

- [ ] 新增 `query_database` 工具（SQLite，只读权限）
- [ ] 新增 `send_notification` 工具（webhook，`needs_permission = ask_user`）
- [ ] 工具 Schema 验证：输入不合法时返回结构化错误，LLM 能根据错误修正参数重试

### 周二 04-28

- [ ] 实现工具并行调用：LLM 一次请求多个工具时，并发执行后合并结果
- [ ] 实现工具调用审计日志：每次工具调用记录时间、参数、结果、耗时
- [ ] 测试 deny 场景：某工具被 Feature Flag 关闭时，LLM 收到什么信息

### 周三 04-29

- [ ] 端到端集成测试
  - 场景1：LLM 请求 `write_file`，用户拒绝，LLM 改用其他方式
  - 场景2：`run_python` 超时，LLM 收到超时错误并调整策略
  - 场景3：关闭 `web_search` flag，LLM 的工具列表里不出现该工具，且行为一致

---

## ResearchAgent MVP 集成（周四~周日）

**目标**：把 Tool System 接入完整的研究助手流程，形成可演示的 MVP。

### 周四 04-30

- [ ] 整合所有工具到 ResearchAgent
- [ ] 实现交互式 CLI：用户输入研究问题 → Agent 自动搜索 → 输出 Markdown 报告

### 周五 05-01

- [ ] 实现报告模板：固定结构（摘要 → 关键发现 → 来源列表）
- [ ] 测试完整研究流程："帮我研究 LangGraph vs LangChain 的架构差异"

### 周六 05-02

- [ ] 性能测试：记录每次研究任务的 Token 用量和耗时
- [ ] 修复发现的 bug（必有）

### 周日 05-03

- [ ] 撰写 MVP README（功能、限制、下一步）
- [ ] 录制一次完整 Demo 运行截图/日志，存档作为 baseline

---

## 阶段1 最终验收

> [!success] 通过标准
> - 新增一个工具只需写一个类，不改 Agent Loop ✓
> - 关闭某工具 flag，LLM 工具列表动态变化 ✓
> - 工具报错 → LLM 自我纠正，不崩溃 ✓
> - `write_file` 执行前终端弹确认，拒绝后优雅降级 ✓
> - 完整研究任务耗时 < 60 秒，Token < 8000 ✓
