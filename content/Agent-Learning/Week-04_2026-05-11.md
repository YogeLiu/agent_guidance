# Week 04 — Planning + LangGraph + Reflection 闭环 + ReAct 对比

← [[Agent-Learning/Week-03_2026-05-04|Week 03]] · [[Agent-Learning/INDEX|总览]] · [[Agent-Learning/Week-05_2026-05-18|Week 05]] →

## 本周目标

Agent 能对复杂任务生成结构化计划并动态修正。引入 LangGraph 实现状态机,把 Prompt 工程化,**完成 Reflection 完整闭环**(in-task critic + cross-task replan 两层),**用数据对比 ReAct vs Plan-Act 两种范式**。

> [!warning] 本周引入 LangGraph
> 学习路径中第一次使用框架。**只用 `StateGraph`**,不引 LangGraph 的 Tool / Memory / Agent 抽象。

> [!info] Reflection 三层不能混
> - **retry**:网络/API 错 → 直接重试(W1 已有)
> - **in-task critic**(W4 周三新增):LLM 输出结果不满足 task description → 在 task 内 self-correct(最多 2 次)
> - **cross-task replan**(W4 周三新增):in-task 修正失败 → 上抛 Replanner 改计划

## Reflection 完整闭环(周三)

### 周三 05-13 — in-task critic + cross-task replan 区分

- [ ] **in-task critic**:每个 task 完成后,LLM 自评 `(task_description, result) → satisfied?`
  - 不满足:把 critic 反馈作为 system message 注入,**在同一 task 内重做(最多 2 次)**
  - Prompt 模板放 `prompts/critic_in_task.md`
- [ ] **cross-task replan**:in-task 修正 2 次仍失败 → 上抛 `Replanner`
  - `planning/replanner.py`(Prompt 从 `prompts/replanner.md` 加载):基于失败原因生成修订计划
  - **防循环**:连续 Replan ≥ 3 次同一任务 → 强制标记"无法完成",继续后续任务
- [ ] **测试用例(关键)**:故意构造一个 task description 是 "总结一篇文章",但 LLM 第一次回复只复述标题
  - 验证:in-task critic 能识别"未达成 task" → 重做并补上总结
  - 验证:**没有立刻 Replan**(否则就是把 retry 当成 reflection 用)
