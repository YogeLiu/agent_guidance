# Week 01 — Agent Loop 从零实现

![[Agent-Learning/INDEX|学习路径总览]]

## 本周目标

不用任何框架，亲手实现一个可运行的 Agent Loop，并搭建工具注册表雏形。

> [!tip] 核心原则
> 这周禁止使用 LangChain / AutoGen / CrewAI。所有代码基于 Anthropic SDK 直接实现。

---

## 阶段0：工具链就绪（周一~周三）

**核心能力**：Tool Calling · 流式输出解析 · 重试逻辑

### 周一 04-20

- [ ] 注册 Anthropic API Key，能跑通第一个 `messages.create` 调用
- [ ] 实现最小 Agent Loop：用户输入 → LLM 调用 → 打印输出
- [ ] 加入流式输出（`stream=True`），用户能看到逐字回复

### 周二 04-21

- [ ] 实现 `web_search` 工具（Tavily API）
- [ ] 实现 `read_file` 工具
- [ ] Agent Loop 检测 `tool_use` block → 执行工具 → 结果注入对话 → 再次调用 LLM

### 周三 04-22

- [ ] 重试逻辑：API 超时 / rate limit 指数退避
- [ ] 工具报错时：错误信息结构化注入对话，LLM 自我纠正
- [ ] 端到端测试："帮我搜索 LangGraph 并总结"全程自动完成

### 阶段0 验收

> [!success] 通过标准
> - web_search 触发 → 结果注入 → LLM 基于结果回答 ✓
> - 工具故意返回错误 → Agent 捕获并在下轮自我纠正 ✓
> - 全程流式输出，无黑盒等待 ✓
> - 代码里 **零** Agent 框架依赖 ✓

## 阶段1 启动：Tool System（周四~周日）

**核心能力**：Tool Permission Model · Pydantic Schema 验证 · Feature Flag 动态工具列表

### 周四 04-23

- [ ] 定义工具统一接口
  - `name` / `description` / `input_schema`（Pydantic）
  - `execute()` / `needs_permission()` → `allow | deny | ask_user`
- [ ] 把 `web_search` 和 `read_file` 重构为新接口

### 周五 04-24

- [ ] 新增 `write_file` 工具（`needs_permission = ask_user`）
- [ ] Human-in-the-Loop：执行前终端弹确认，拒绝后 Agent 优雅处理

### 周六 04-25

- [ ] 新增 `run_python` 工具（subprocess + timeout 沙箱）
- [ ] 实现 `ToolRegistry`：`register(tool)` + `get_tools_for_llm(flags)` 动态返回列表

### 周日 04-26

- [ ] 实现 Feature Flag：`write_file=False` 时工具列表里不出现该工具
- [ ] 整体联调：用户输入 → 工具执行全链路
- [ ] 写 README 记录工具接口规范
