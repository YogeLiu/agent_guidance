## 本周目标

完成工业化(Cost / Trace / 沙箱 / 并发硬化),用 **CodeReviewAgent**(2 天横向迁移)证明基础设施可复用,用 **DataAnalysisAgent**(90 分钟 Spike)验证抽象层通用,跑 Eval 全集 + 5 道自测题。这是 6 周学习的最终交付。

## 第二项目:CodeReviewAgent(周三~周四,横向迁移验证)

> [!info] 强约束
> **不准改 core/ planning/ agents/ evals/ observability/** —— 只能新增:

### 周三 05-27 — 设计 + 工具 + Prompt

- [ ] 项目设计
  - 输入:本地 git diff(`git diff main..HEAD`)或 PR URL
  - 输出:Markdown 格式 code review 报告(类别:bug / style / suggestion / 赞美)
  - 路径选 single agent(W5 已证伪 multi-agent 必要性)或复用 Coordinator + ReviewerWorker
- [ ] 实现 `tools/git_diff.py`:接 git CLI 取 diff,按文件分块返回
- [ ] 实现 `tools/lsp_check.py` 占位
- [ ] 写 `prompts/code_review/system.md` + `prompts/code_review/template.md`
- [ ] 写 `main_codereview.py`(入口)

### 周四 05-28 — 跑通 + 验证 + 复用率报告

- [ ] 跑 5 个真实 PR diff(从 GitHub trending repo 拉,或自己历史 PR),输出 review 报告
- [ ] **写 5 条 CodeReview eval case** 进 `evals/cases_codereview.jsonl`,LLM-as-Judge 评分,目标 ≥ 75
- [ ] **写复用率报告 `docs/transfer_report.md`**
  ```
  原 ResearchAgent 代码总行数:XXXX
  CodeReviewAgent 新增行数:XXX(必须 < 500)
  复用率:XXXX / (XXXX + XXX) = XX%(必须 ≥ 80%)
  改动到 core/ 等"禁区"的行数:XX(必须 = 0)
  ```
- [ ] 通过标准:**复用率 ≥ 80%,新增 < 500 行,evals Judge 分 ≥ 75**
- [ ] 不通过 → 写"基础设施哪里被场景绑死了",这本身是重要学习产出
