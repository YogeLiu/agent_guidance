## Observability（周二）

### 周二 05-26

- [ ] 实现 Trace/Span 模型

```
Session
  └── Plan
        ├── Coordinator Call (span)
        ├── Worker: Searcher-1 (span)
        │     └── Tool: web_search (span)
        ├── Worker: Searcher-2 (span)
        └── Worker: Writer (span)
```

- [ ] 每个 Span 记录：开始时间、结束时间、Token 消耗、输入输出摘要
- [ ] 实现 Trace 导出：JSON 格式，可用 LangSmith 或本地查看
- [ ] 实现简单 CLI 可视化：`researchagent trace show <session_id>`

## 系统集成（周三~周四）

### 周三 05-27

- [ ] 打通完整流程：
  ```
  用户输入
    → Memory 注入（recall 相关记忆）
    → Planner 生成计划
    → Coordinator 执行（Searcher + Analyst + Writer Workers）
    → Memory 更新（记录本次研究结论）
    → 输出报告 + 成本摘要
  ```
- [ ] 确保 Memory 正确传递给 Coordinator（但 Worker 不感知 Memory）

### 周四 05-28

- [ ] 实现 Session 管理：每次对话有唯一 Session ID，完整日志可追溯
- [ ] 实现错误边界：任意一层（Memory / Planning / Worker）失败，系统能降级而不崩溃
  - Worker 失败 → Coordinator Replan
  - Memory 读写失败 → 继续运行，记录警告
  - Planning 失败 → 降级为单 Agent 模式
